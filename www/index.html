loggy-node

loggy-node is a test application for Cloud Foundry that demonstrates how to deal with logging in node.js.

Some things you can do to the server, all of which will kill it:

have the process exit with status 0

have the process exit with status 1

throw an uncaught exception

consume all memory

Don't worry! Cloud Foundry will restart the app, after it dies a horrible death.

The code for this app is available at https://github.com/IBM-Bluemix/loggy-node. The article in the project contains more information about using logging in your Cloud Foundry applications, and a local copy of the article is available here.

You can install your own copy of this app on Bluemix, by clicking the Deploy to Bluemix button, below.
